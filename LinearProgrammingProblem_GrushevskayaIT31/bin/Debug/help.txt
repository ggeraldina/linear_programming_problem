О ПРОГРАММЕ

Программа предназначена для решения задачи линейного программирования.

 

----------
ОБЗОР МЕНЮ
----------
МЕНЮ/ФАЙЛ/МЕТОД

Функционал программы предлагает 2 подхода к решению задачи
1)	с помощью метода искусственного базиса 
(Для решения данным методом в меню следует выбрать: Файл/Метод/Решить с помощью метода искусственного базиса);
2)	с использованием заданных базисных переменных
(В меню выбрать: Файл/Метод/Решить с использованием заданных базисных переменных).
При запуске программы по умолчанию выбрано решение с помощью метода искусственного базиса.

Более подробно оба подхода рассмотренны в соответствующих пунктах справки.

---
МЕНЮ/ФАЙЛ/ОТКРЫТЬ

После нажатия пункта меню Файл/Открыть предоставляется возможность загрузить ранее сохраненную задачу.

---
МЕНЮ/ФАЙЛ/СОХРАНИТЬ 
(это отладочный пункт меню, который в итоговой версии отсутствует)

После нажатия пункта меню Файл/Сохранить текущая задача будет сохранена в папке tasks, расположенной в той же директории что и программа, под именем task_<текущая дата>_<текущее время> с уведомлением об успешности сохранения и указанием имени сохраненной задачи. 

---
МЕНЮ/ФАЙЛ/СОХРАНИТЬ КАК

Если есть желание сохранить текущую задачу под произвольным именем и/или в определенной директории, то тогда следует выбрать в меню Файл/Сохранить как. После этого в диалоговом окне предоставится возможность самому указать имя для текущей задачи и ее месторасположение. По умолчанию программа предлагает сохранить задачу в папке tasks, расположенной в той же директории что и программа, под именем task_<текущая дата>_<текущее время>.

---
МЕНЮ/СПРАВКА/О ПРОГРАММЕ

В меню Справка/О программе можно найти краткую информацию о назначении программы, а также информацию об авторе.

---
Метод искусственного базиса

Для решения задачи методом искуственного базиса в меню следует выбрать Файл/Метод/Решить с помощью метода искусственного базиса. 

Далее нужно ввести задачу. Это можно сделать 2 способами.

1)	загрузить ранее сохраненную задачу (Для этого нужно в меню выбрать Файл/Открыть);
2)	ввод задачи вручную
	2.1)	Неоходимо указать количество используемых переменных. Для этого нужно выбрать соответствующее число в выпадающем списке, расположенном рядом с фразой "число переменных".
	2.2)	После следует сообщить программе количество ограничений задачи, т.е. выбрать соответствующее число рядом с фразой "количество условий". Количество условиий не может быть больше числа переменных. 
	2.3)	Затем нужно ввести коэффициенты целевой функции и коэффициенты ограничений задачи. О принимаемых программой форматах чисел (коэффициентов) можно узнать в соответствующем пункте справки.
	*)	Примечание: Важно ввести задачу в указанной ранее последовательности, так как максимально возможное количество условий зависит от числа переменных. А количество строк для ввода ограничений задачи зависит от введенного количества условий.

Решение задачи с помощью метода искуственного базиса разделено на 2 части:
I Часть)	поиск базиса с помощью метода искуственного базиса;
II Часть)	решение задачи симплекс методом.

После ввода задачи следует перейти к решению I Части. Для этого нужно нажать кнопку "Решить I часть". Если проверка введенных условий будет пройдена успешно, то тогда на экране будет отображена симплекс-таблица, в противном случае будет выведено сообщение об ошибке.

Программа поддерживает 2 режима решения задачи:
1)	пошаговый, с возможностью выбора опорного элемента;
2)	автоматический.
Данные режимы совмещены. Для решения задачи в пошагововом режиме достаточно выбрать любой из возможных опорных элементов. Для решения задачи автоматически нужно нажать "рассчитать автоматически", тогда программа сама продолжит поиск базиса, начиная с отображаемой в данный момент симплекс-таблицы.
С помощью кнопок навигации, расположенных под таблицей, можно просмотреть предыдущие этапы решения. В методе искусственного базиса представлены 2 типа чередующихся друг за другом таблиц: таблица с возможностью выбора опорного элемента и "промежуточная" таблица с еще невычеркнутым столбцом и выделенным серым цветом местом расположения выбранного ранее опорного элемента.
